# BOOT_LOOP_BLACK_SCREEN_FIX_V12

Срочное исправление:
- добавлен флаг sessionStorage `baraholka_boot_shown_v12`, чтобы Telegram/WebView не показывал загрузочный экран повторно после внутреннего reload;
- убран React StrictMode, чтобы исключить удвоение эффектов в окружениях, где приложение может запускаться не production-режимом;
- добавлен ErrorBoundary: при любой ошибке React больше не будет чёрного экрана, появится экран восстановления;
- стартовые сетевые ожидания сокращены;
- сохранён прежний вид progress bar: обычное плавное изменение ширины без неудачного интервала;
- fullscreen request по-прежнему отключён;
- опасные страницы из старой sessionStorage не восстанавливаются.

Деплой:
- нужен redeploy на Vercel;
- Render не нужен.
